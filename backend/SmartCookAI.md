# 产品需求文档 (PRD) - SmartCook AI

| 文档版本 | V1.0 |
| :--- | :--- |
| **产品名称** | SmartCook AI (智能食谱生成器) |
| **项目类型** | Web 应用 (移动端适配) |
| **优先级** | P0 (最高优先级) |
| **撰写人** | AI 产品经理 |

---

## 1. 项目背景与价值
### 1.1 痛点分析 (Problem)
*   **决策瘫痪**：用户面临“不知吃什么”的世纪难题。
*   **食材浪费**：冰箱剩余零散食材（如半个洋葱、200g 鸡肉）缺乏合适的烹饪方案。
*   **门槛过高**：传统菜谱搜索需要精确菜名，不支持基于现有食材的反向推导。

### 1.2 产品目标 (Goal)
*   **核心价值**：通过 AI 将有限/剩余食材转化为创意且可执行的美味食谱。
*   **交付物**：轻量级 Web 应用，实现“输入食材 -> 生成食谱 -> 辅助烹饪/采购”的全流程闭环。

---

## 2. 用户角色 (User Personal)
*   **家庭主掌勺**：需要处理家庭剩余食材，追求营养搭配。
*   **独居青年/学生**：食材储备少，烹饪技能有限，追求简单快捷。
*   **健身/特殊饮食者**：对热量、营养成分有明确要求（对应进阶需求）。

---

## 3. 功能需求说明 (Functional Requirements)

### 3.1 核心业务流程
`输入/选择食材` -> `设置筛选条件 (可选)` -> `AI 生成推荐` -> `查看详情` -> `生成清单/开始烹饪`

### 3.2 详细功能模块

#### 模块一：食材输入系统 (P0 - Must Have)
*   **用户故事**：作为用户，我希望能快速录入冰箱里现有的东西，不需要一个个打字。
*   **功能细节**：
    1.  **分类选择面板**：提供 ICON 形式的快捷分类（蔬菜、肉禽、海鲜、主食、调料）。
    2.  **智能搜索栏**：支持模糊搜索（如输入“红”提示“红萝卜、红薯”）。
    3.  **自由文本输入**：允许用户输入长尾食材（如“老干妈”、“昨天剩下的烤鸭”）。
    4.  **量词与状态**：支持结构化输入（例：`鸡肉` + `200g` + `冷冻`）。
    5.  **常用库**：本地缓存用户的“常备食材”，支持一键添加。

#### 模块二：AI 智能食谱生成引擎 (P0 - Must Have)
*   **用户故事**：作为用户，我希望 AI 能根据我给的几样东西，像大厨一样告诉我能做什么，而不是仅仅搜索数据库。
*   **功能细节**：
    1.  **生成逻辑**：调用 LLM（大语言模型）API，单次生成 3-5 个方案。
    2.  **菜谱结构卡片**：
        *   **创意菜名**：生成具有吸引力的名称（如“黄金满屋蛋炒饭”而非“蛋炒饭”）。
        *   **食材状态**：明确标注 **[已有]** 和 **[需补充]** 的食材。
        *   **难度/时间/热量**：结构化标签展示。
    3.  **合理性校验**：Prompt 需包含逻辑约束，避免生成“西瓜炒月饼”等黑暗料理（除非用户要求）。

#### 模块三：筛选与排序 (P0 - Must Have)
*   **功能细节**：
    *   **菜系维度**：中/西/日韩/东南亚。
    *   **口味维度**：酸/甜/苦/辣/咸/清淡。
    *   **场景维度**：早餐/快手菜（15min）/硬菜（1h+）。
    *   **技能维度**：有手就行（新手）/ 挑战自我（大厨）。

#### 模块四：菜谱详情与交互 (P0 - Must Have)
*   **功能细节**：
    1.  **分步指导**：步骤条清晰展示，支持勾选“已完成”。
    2.  **收藏夹**：支持创建自定义分组（如“减脂餐”、“周末大餐”）。
    3.  **历史记录**：自动保存最近 20 次生成记录。
    4.  **导出功能**：生成长图（包含食材表+步骤）保存至相册。

#### 模块五：智能扩展功能 (P1 - Should Have / 进阶挑战)
*   **智能替代**：
    *   逻辑：检测缺失食材 -> 查询向量数据库 -> 推荐风味/口感相似食材（如：缺柠檬汁 -> 推荐白醋）。
    *   交互：在菜谱详情页高亮显示“可用 [白醋] 替代”。
*   **购物清单生成**：
    *   逻辑：提取菜谱中[需补充]的食材 -> 聚合 -> 按超市分区分类。
    *   交互：支持手动增删改，支持一键复制文本。

#### 模块六：数据与个性化 (P2 - Could Have / 进阶挑战)
*   **营养分析**：接入营养数据库，计算卡路里及宏量营养素（碳水/蛋白/脂肪），提供过敏原警示。
*   **智能优化**：
    *   **余料推荐**：根据本次做菜剩下的食材，预测明日菜谱。
    *   **成本优化**：优先推荐使用当季低价食材的菜谱。

---

## 4. 非功能需求 (Non-functional Requirements)
1.  **响应速度**：AI 生成过程需有 Loading 动画或趣味文案（如“AI 正在切菜...”），首字生成时间 < 3秒，完整生成 < 15秒。
2.  **并发性能**：支持 1000 QPS（每秒查询率），确保饭点高峰期不崩溃。
3.  **可用性**：界面需适配移动端浏览器（Responsive Design）。

---

## 5. 数据埋点与考核指标 (Metrics)
*   **核心指标**：
    *   **食谱采纳率**：用户生成后点击进入详情页的比例。
    *   **完读率**：用户在详情页停留超过 30秒 或点击“收藏/导出”的比例。
*   **业务指标**：
    *   **复购率（留存）**：次周留存率。
    *   **分享率**：生成的菜谱图片被保存或分享的次数。
